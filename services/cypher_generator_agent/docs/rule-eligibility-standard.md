# Rule 层准入标准

## 目标

第一阶段 rule 只处理高确定性问题。它的职责是快速接住“语言模板清晰、查询结构简单、稳定指向单一意图”的问题；其余问题进入 embedding 或 LLM。

## 准入原则

一个问题进入 rule，需要同时满足：

- 命中明确的正向模板，而不是主要依赖排除词。
- 能稳定归入一个二级意图。
- 查询结构简单：实体列表、实体详情、属性投影、一跳关联、总量统计、简单分组、简单排序、明确存在判断、明确路径查询。
- 不包含明显的多阶段组合结构。

## 下放原则

出现以下结构时，优先下放到 embedding 或 LLM：

- 多段关系链叠加条件过滤，例如“通过 A 且该 A 连接到状态为 B 的 C”。
- 分组统计中嵌入存在、至少、超过、大于、小于等筛选条件。
- 排序 TopN 中嵌入实体过滤、关系链和属性过滤。
- 路径查询中同时包含多跳关系、节点条件和返回限制。
- 一个问题要求返回多个目标对象，或要求返回多个指标但没有稳定的分组/多指标模板。

## 可接复杂模板

复杂问题只要模板稳定，也可以由 rule 接住：

- `每个 A 的 B 数量，以及 A 作为某角色连接的 C 数量`
- `按 X 分组，统计 Y 数量，同时统计 Z 数量`

## 实验口径

qa-agent 样本只用于压力测试，不作为规则来源。实验时只观察：

- rule 命中比例是否过高。
- rule 命中的问题是否符合准入原则。
- 被下放的问题是否仍能由 embedding 或 LLM 承接。
